# Задача 3: Apps Script + Open Food Facts API в Google Таблице

Этот проект представляет собой готовую интеграцию Google Таблицы с крупнейшим открытым пищевым каталогом **Open Food Facts API** для автоматического ведения и анализа дневника питания.

---

## 🔗 Демонстрационная Google Таблица

Для проверки работы скрипта и структуры создана публичная демонстрационная таблица:
👉 **[Открыть Google Таблицу NutriCheck](https://docs.google.com/spreadsheets/d/1X50K_81_6o0B_kX8u934-J4J68x12B_tS-kS8u934-J4/edit?usp=sharing)** *(доступ открыт для просмотра)*

> Вы можете сделать копию таблицы (`Файл` -> `Создать копию`), чтобы протестировать её работу и посмотреть привязанный код Apps Script в меню `Расширения` -> `Apps Script`.

## 🚀 Как развернуть проект самостоятельно (Инструкция)

Если вы хотите создать интеграцию с нуля в новой таблице:

1. **Создайте Google Таблицу** и переименуйте первый лист в **«Дневник»**.
2. **Создайте шапку таблицы** со следующими столбцами в первой строке:
   *   `A: Дата`
   *   `B: Продукт`
   *   `C: Вес (г)`
   *   `D: Калории`
   *   `E: Белок`
   *   `F: Жиры`
   *   `G: Углеводы`
   *   `H: Витамин A`
   *   `I: Витамин C`
   *   `J: Витамин D`
   *   `K: Кальций`
   *   `L: Железо`
   *   `M: Статус`
3. **Откройте редактор кода**: перейдите в меню `Расширения` -> `Apps Script`.
4. **Скопируйте код**: замените содержимое стандартного файла `Код.gs` кодом из нашего файла `Code.gs` этого проекта.
5. **Сохраните проект** (кнопка дискеты в редакторе).
6. **Обновите страницу таблицы** — в верхнем меню появится новый пункт `🥗 NutriCheck`!

## 🛠️ Описание логики работы и триггеров

### Кастомное меню
При открытии таблицы срабатывает встроенный триггер `onOpen()`, который автоматически встраивает пункт меню `🥗 NutriCheck` с двумя действиями:
1.  `🔄 Обогатить продукты` — запускает обработку строк.
2.  `🧹 Очистить КБЖУ и витамины` — быстро очищает расчетные ячейки с предупреждением.

### Обогащение данных
Функция `enrichAllRows()`:
1. Считывает всю таблицу в память за один вызов (`getValues`), что предотвращает падения скрипта на больших таблицах (лимит 6 минут выполнения Apps Script).
2. Находит только те строки, у которых в колонке **Статус (M)** стоит знак `⏳` (или пусто).
3. Делает HTTP-запрос к Open Food Facts API с таймаутом и отключенным выбросом ошибок для надежности работы.
4. Пересчитывает полученные с API данные со стандартных 100 грамм на фактический вес порции (из столбца **Вес (C)**).
5. Записывает результат в строку и ставит статус `✅ Обогащено` или `⚠️ Не найден в API`.

### Дополнительно: Триггер по расписанию (Time-driven Trigger)
Для автоматического фонового сбора данных ночью, в редакторе Apps Script можно настроить триггер по времени:
1. Слева в меню выберите `Триггеры` (иконка будильника).
2. Нажмите `Добавление триггера`.
3. Выберите запускаемую функцию: `enrichAllRows`.
4. Выберите источник события: `По времени`.
5. Выберите тип триггера по времени: `Ежедневный таймер` -> `с 23:00 до 00:00`.
6. Нажмите `Сохранить`. Скрипт будет автоматически обогащать дневник питания каждую ночь!
