# NutriCheck — Сценарий и инструкция для проведения Live-Демо

Пошаговый сценарий демонстрации экосистемы **NutriCheck** для технического интервью в компанию РЕДАНТ.

Сценарий показывает сквозной поток данных (Data Flow):
`Google Таблица (Ввод) → Python CLI (Аналитика) → Web Dashboard (Визуализация)`

> **История для интервьюера**: «Представьте нашего пользователя Алексея. Он активно занимается спортом, следит за здоровьем и хочет понять — получает ли он достаточно витаминов из повседневной еды, или его рациону требуются добавки. Мы покажем, как NutriCheck решает эту задачу на трёх уровнях».

## 📍 Этап 1: Google Таблица и Apps Script (Сбор данных)
*Показываем, как удобно и быстро собирать лог питания в привычном интерфейсе.*

1. **Перейдите** на вкладку браузера с Google Таблицей (открыта на Шаге 0.3), лист **«Дневник»**.
2. **Покажите кастомное меню**: укажите интервьюеру на верхнее меню `🥗 NutriCheck` (рядом с «Справка»).
3. **Введите 2–3 строки** в конец таблицы. Кликните в первую пустую ячейку столбца **A** и введите по столбцам (между ячейками — клавиша `Tab`). Используйте **проверенные брендовые товары** — они гарантированно есть в базе с полным составом:
   | A — Дата | B — Продукт | C — Вес (г) | M — Статус |
   |----------|-------------|-------------|------------|
   | `25.05.2026` | `Chocapic` | `50` | `⏳` |
   | `25.05.2026` | `Frosties Kellogg's` | `40` | `⏳` |
   | `25.05.2026` | `Crunch Nestle` | `50` | `⏳` |

   > ⚠️ **Не используйте сырые/русские названия** (Шпинат, Курица, Гречка) — Open Food Facts это база упакованных товаров, и по таким запросам вернётся мусор или нули. Список других надёжных товаров: `Cheerios`, `Lion Cereal`, `Fitness Nestle`.
4. **Запустите скрипт**: в меню выберите `🥗 NutriCheck` → `🔄 Обогатить продукты`.
   - *Если появится запрос авторизации* (при первом запуске): нажмите `Продолжить` → выберите аккаунт → `Дополнительные настройки` → `Перейти к проекту (небезопасно)` → `Разрешить`.
5. **Наблюдаемый эффект** (комментируйте вслух):
   - Ячейка статуса меняется на `⏳ Поиск...`.
   - Через 1–2 секунды столбцы D–L заполняются: калории, БЖУ, витамин D, кальций, железо — пересчитанными под указанный вес.
   - Статус меняется на `✅ Обогащено`.
   - *(Примечание: витамины A и C в базе заполнены редко — их нули это норма для источника данных, а не ошибка скрипта.)*
6. **Реплика**:
   > «Скрипт оптимизирован под лимиты Google Apps Script: вместо построчной записи мы считываем всю таблицу в память одним вызовом `getValues()`, обрабатываем и записываем пакетно. Это предотвращает таймаут выполнения в 6 минут на больших дневниках».

## 📍 Этап 2: Python CLI Анализатор (Глубокая аналитика)
*Показываем консольную аналитику, интеграцию с API и генерацию отчётов.*

1. **Откройте новое окно PowerShell** и перейдите в папку анализатора:
   ```powershell
   cd D:\Projects\Nutricheck\task2-analyzer
   ```
2. **Запустите автотесты** (демонстрирует техническую культуру) — скопируйте команду:
   ```powershell
   py -m unittest tests/test_analyzer.py
   ```
   Ожидаемый результат внизу вывода:
   ```
   Ran 4 tests in 0.0XXs
   OK
   ```
3. **Запустите аналитический скрипт** на готовом дневнике из 20 продуктов:
   ```powershell
   py main.py --input data/sample_food_log.csv --gender male --age 32 --activity moderate
   ```
4. **Наблюдаемый эффект** (комментируйте вслух):
   - Идёт лог поиска продуктов в Open Food Facts с прогрессом `[1/20]`, `[2/20]`, ...
   - Выводится цветная таблица (через `colorama`): КБЖУ, витамины, ASCII-шкалы покрытия норм RDA.
   - Внизу — карточки выявленных дефицитов и сообщение, что отчёты сохранены в `output/`.
5. **Покажите сгенерированные отчёты** — выполните, чтобы открыть папку в Проводнике:
   ```powershell
   ii output
   ```
   Откройте `report.json` и `report.csv`, покажите структуру. **JSON понадобится на Этапе 3.**
6. **Реплика**:
   > «В Python-анализаторе три enterprise-практики: in-memory кэш запросов (повторяющийся продукт запрашивается у API один раз), retry с экспоненциальной задержкой при сетевых сбоях и 503, и генерация структурированного JSON-отчёта, готового к импорту во фронтенд».

## 📍 Этап 3: Web Dashboard (WOW-эффект и визуализация)
*Показываем адаптивный интерфейс, Canvas-графику и интерактивность.*

1. **Откройте браузер** на странице **http://localhost:3000** (сервер уже запущен на Шаге 0.4).
   - *Если сервер не запущен*: в новом окне PowerShell выполните `cd D:\Projects\Nutricheck\task1-web; node server.js`.
2. **Задайте профиль** вверху справа: Пол — `Мужской`, Возраст — `32`, Активность — `Средняя`.
3. **Покажите интерактивный поиск**:
   - В поле «🍎 Добавить продукты» введите `Морковь` — подождите ~0.5 сек (debounce), появится выпадающий список из API.
   - Кликните на первый результат — продукт добавится в список.
   - Повторите для `Шпинат` и `Лосось`.
4. **Нажмите кнопку `⚡ Рассчитать нутриенты`** — радар-диаграмма перерисуется, появятся прогресс-бары и рекомендации.
5. **Покажите WOW-эффект — Импорт JSON из Этапа 2**:
   - Нажмите кнопку `📊 Импорт JSON` (вверху левой колонки).
   - В окне выбора файла вставьте в строку пути:
     `D:\Projects\Nutricheck\task2-analyzer\output\report.json`
   - Нажмите `Открыть`.
6. **Наблюдаемый эффект** (комментируйте вслух):
   - Дашборд мгновенно заполняется всеми 20 продуктами из дневника.
   - Полигон радар-диаграммы разворачивается из центра — покрытие 8 витаминов.
   - Прогресс-бары показывают статусы; дефицитные нутриенты подсвечиваются красным.
   - Снизу генерируются карточки рекомендаций с продуктами.
7. **Запустите браузерные автотесты** — откройте в новой вкладке:
   **http://localhost:3000/test-runner.html**
   - Покажите зелёную плашку `🎉 Все тесты пройдены успешно!`.
8. **Реплика**:
   > «Веб-интерфейс — чистый ванильный JS и CSS без тяжёлых библиотек. Радар-диаграмма отрисована на HTML5 Canvas с тригонометрией и анимирована через `requestAnimationFrame`. Приложение лёгкое, быстрое и не зависит от версий фреймворков».

## 🧯 Если что-то пошло не так

| Симптом | Причина | Решение |
|---------|---------|---------|
| «Продукт не найден в API» (веб/CLI) | Нет интернета или сбой Open Food Facts | Проверьте сеть, повторите. Для веба используйте уже готовый импорт `report.json` |
| `node server.js` → ошибка `EADDRINUSE` | Порт 3000 занят | Закройте старое окно сервера или смените `PORT` в начале `task1-web/server.js` |
| `ModuleNotFoundError` при `py main.py` | Не установлены зависимости | `cd D:\Projects\Nutricheck\task2-analyzer; py -m pip install -r requirements.txt` |
| `py` не распознан | Используется другая команда Python | Замените `py` на `python` во всех командах |
| Меню `🥗 NutriCheck` отсутствует в Таблице | Не та таблица / не обновлена страница | Обновите вкладку (`F5`); проверьте, что открыта демо-таблица из README |
| `http://localhost:3000` не открывается | Сервер не запущен | Выполните Шаг 0.4 |
