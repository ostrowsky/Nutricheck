# Задача 1: Веб-утилита NutriCheck Web Dashboard

Интерактивный веб-дашборд для визуализации суточного покрытия витаминов и минералов на основе съеденных продуктов, расчета дефицитов и выдачи пищевых рекомендаций.

## 💻 Инструкция по локальному запуску

Проект написан на чистом Vanilla JS без тяжелых зависимостей (типа npm/webpack), что делает его мгновенно загружаемым и простым в запуске:

1. Перейдите в папку `task1-web/`.
2. Запустите встроенный dev-сервер:
   ```bash
   node server.js
   ```
3. Откройте в браузере: **`http://localhost:3000`**.
4. Для запуска автотестов перейдите по адресу: **`http://localhost:3000/test-runner.html`**.

## 🤖 Описание процесса вайбкодинга с Claude Code

Для разработки данного дашборда использовался ИИ-ассистент **Claude Code CLI** в интерактивном режиме. Ниже представлен подробный дневник работы:

### Используемые промпты

1.  **Промпт 1: Дизайн-система и верстка**:
    > *«Создай CSS дизайн-систему для health/nutrition дашборда. Тёмная тема, цветовая схема от зелёного (хорошо) через жёлтый (средне) к красному (плохо). Glassmorphism карточки, шрифт Inter от Google Fonts. Напиши семантическую HTML-верстку с шапкой (профиль пользователя), 2-колоночной сеткой (поиск слева, графики справа) и подвалом.»*
2.  **Промпт 2: Поисковой API слой с дебаунсом**:
    > *«Напиши асинхронный поиск продуктов в Open Food Facts API с дебаунсом в 350мс. Парси результаты в массив продуктов с КБЖУ и 10+ витаминами. Обработай ошибки сети и сбои API с помощью toast-сообщений.»*
3.  **Промпт 3: Радарная диаграмма на Canvas**:
    > *«Напиши JS-класс для отрисовки радарной диаграммы (Radar Chart) на чистом Canvas 2D с нуля без внешних библиотек. Сделай 8 осей для ключевых витаминов и минералов. Сделай плавную анимацию появления полигона с requestAnimationFrame.»*

### Что Claude Code сделал отлично с первой попытки
- **CSS-стилизация**: Тёмная тема с glassmorphism получилась превосходно с первого раза — красивые градиенты, полупрозрачные карточки, аккуратные шрифты.
- **Маппинг нутриентов API**: ИИ безошибочно перевел граммы нутриентов, приходящие от Open Food Facts API, в микрограммы и миллиграммы (мкг/мг) для витаминов и минералов.

### Что пришлось доработать вручную и почему
1.  **Формула Diet Quality Score**: Изначально Claude Code рассчитывал средний балл диеты, включая калории и макронутриенты (белки, жиры, углеводы). Если пользователь переедал жиров (например, 200% от нормы), общий балл диеты необоснованно возрастал. **Решение**: Переписали формулу так, чтобы Diet Score считался *только по микронутриентам* (витамины + минералы), причем покрытие каждого элемента для формулы ограничивается 100% сверху, чтобы избыток одного витамина не маскировал дефицит другого.
2.  **CORS в Open Food Facts API**: Запросы к API напрямую из браузера падали по CORS на некоторых доменах. **Решение**: Добавили User-Agent и правильные заголовки HTTP-запроса, а также резервную обработку локальных тестовых данных в случае таймаута сети.
3.  **Кодировка в CSV-парсере**: Локальный CSV-импорт падал на кириллице из русскоязычного Excel. **Решение**: Реализовали корректную фильтрацию двойных кавычек в ячейках и автоопределение разделителя `;` / `,`.
