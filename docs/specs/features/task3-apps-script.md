# Спецификация: NutriCheck Google Sheets Integration (Task 3)

## Цель продукта
Интеграция Google Таблицы с базой данных Open Food Facts посредством Google Apps Script для автоматического ведения и обогащения дневника питания.

## Функциональные требования (User-visible behavior)
1. **Пользовательское меню**: В интерфейс Google Таблицы при открытии (`onOpen`) динамически добавляется кастомное меню `🥗 NutriCheck` с кнопкой `🔄 Обогатить продукты`.
2. **Обогащение строк**: При запуске скрипт сканирует лист «Дневник», находит строки со статусом `⏳` (ожидает обработки), считывает название продукта, запрашивает API Open Food Facts через `UrlFetchApp`, и заполняет столбцы:
   - Калории, Белки, Жиры, Углеводы, Витамин A, Витамин C, Витамин D, Кальций, Железо.
   - Значения пересчитываются со 100 грамм базы данных на фактический вес порции (колонка «Вес»).
3. **Обновление статуса**: После успешной обработки статус меняется на `✅ Обогащено`. В случае ошибки (продукт не найден) — статус меняется на `⚠️ Не найден`.
4. **Автоматический триггер по времени**: Скрипт может запускаться по триггеру (например, каждую ночь в 23:00) для фоновой обработки за день.

## Инварианты данных (Invariants)
- Скрипт не должен перезаписывать строки, которые уже имеют статус `✅ Обогащено` (защита от лишнего трафика и перезаписи ручных правок).
- Обработка ошибок: Любые сбои сети или API пишутся в отдельный столбец статуса или в отдельный лист «Лог», предотвращая прерывание скрипта на середине таблицы.

## Граничные случаи (Edge Cases)
- **Превышение лимита выполнения Apps Script (6 минут)**: Скрипт должен обрабатывать строки пачками или завершаться корректно при угрозе таймаута.
- **Пустые строки в таблице**: Пропускаются без генерации ошибок.
